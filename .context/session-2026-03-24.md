## Changes Made

### Dual-Backend Transcriber Architecture (commit 4fe18a4)

Implemented abstract `Transcriber` interface supporting multiple backends:

**New Files Created:**
| File | Purpose |
|------|---------|
| `internal/transcriber/transcriber.go` | Interface definitions (Transcriber, Stream, Backend) |
| `internal/transcriber/moonshine_adapter.go` | Wraps existing Moonshine implementation |
| `internal/transcriber/whisper_adapter.go` | Whisper.cpp implementation (build-tagged) |
| `internal/transcriber/whisper_stub.go` | Stub when whisper not compiled |
| `internal/transcriber/factory.go` | Backend selection factory |

**Config Additions:**
- `BACKEND=moonshine|whisper` - Select transcription backend
- `WHISPER_MODEL=<path>` - Path to Whisper GGML model
- `THREADS=N` - Thread count for Whisper (1-32)
- `LANGUAGE=de|en|...` - Language code for Whisper

**Security Hardening in Whisper Adapter:**
- Sample rate validation (8kHz-192kHz)
- Language whitelist validation (prevents injection)
- Bounded audio buffer (30s max) prevents memory exhaustion
- Transcription timeout (60s)
- Input size limits on PCM chunks

**Language Support:**
- Moonshine: en, es, ar, ja (native streaming)
- Whisper: 40+ languages including German (de)

### Expanded Filler Removal (commit 998f4c4)

Categorized filler patterns for comprehensive coverage:

**Hesitation Sounds (8 patterns):**
- `um+`, `uh+`, `ah+`, `oh+`, `eh+` (extended variants)
- `er`, `erm`, `err` (British)
- `hm+`, `mhm+` (thinking/listening)

**Agreement Sounds (4 patterns):**
- `uh-huh`, `uh huh` (affirmative)
- `uh-uh`, `nuh-uh` (negative)
- `mm-hm`, `mm-hmm`

**Discourse Markers (6 patterns):**
- `you know`, `I mean`, `I guess`, `you see`
- `sort of`, `kind of`

**Technical Improvements:**
- Pattern ordering: compound first to prevent partial matches
- Punctuation cleanup after filler removal
- Word boundaries prevent false positives (album, wanderer)

## To Use German (Whisper Backend)

1. Download Whisper model: `ggml-base.bin` or `ggml-small.bin`
2. Configure `~/.config/moonshine/config`:
   ```
   BACKEND=whisper
   WHISPER_MODEL=/path/to/ggml-base.bin
   LANGUAGE=de
   THREADS=4
   ```
3. Rebuild with `-tags whisper` flag

### Tray Language/Backend Switching (commit ba090a0)

Added system tray menu option to switch between English (Moonshine) and German (Whisper) backends:

**New Tray Menu:**
- Language submenu with radio-style selection (●/○)
- "English (Moonshine)" - Fast local English transcription
- "German (Whisper)" - German transcription via Whisper

**Implementation:**
- `Config.Save()` - Persists config changes to disk atomically (temp + rename)
- `Daemon.GetBackend()` / `GetLanguage()` - Accessors for tray
- `Daemon.SetBackendConfig(backend, language)` - Updates config and saves
- `Tray.syncLanguageChecks()` - Updates radio indicators

**Behavior:**
- Selection updates `~/.config/moonshine/config` with `BACKEND=` and `LANGUAGE=`
- Shows notification: "Switched to [language]. Restart daemon to apply."
- Daemon restart required since transcriber initializes at startup

### Configurable Sentence-End Punctuation (commit 612e5f0)

Added `SENTENCE_END` config option to customize default punctuation:

| Value | Result |
|-------|--------|
| `.` (default) | Adds period |
| `` (empty) | No punctuation |
| `none` | No punctuation |
| `!` or any char | Uses that character |

Files modified:
- `internal/config/config.go` - Added `SentenceEnd()` method
- `internal/daemon/daemon.go` - Updated `autoPunctuation()` to accept parameter
